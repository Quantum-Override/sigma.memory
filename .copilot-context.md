# Document Standards — sigma.memory
- Standards: [q-or/DOC_STANDARDS.md](../q-or/DOC_STANDARDS.md)
- Registry:  [q-or/registry.aml](../q-or/registry.aml)
- This project's doc root: `sigma.memory/docs/` (backlog.md, roadmap.md, design.md)
- Cross-project link syntax: `../other-project/subdir/filename.md`
- To publish a doc for cross-project use: add entry to registry.aml first
- ID prefix for new bugs: `BR-{YYMM}-sigma-memory-{seq:03d}`
- Registered backlog: [BL-sigma-memory](docs/backlog.md)
- **v0.2.x archive:** [`../archive/sigma.mem_0.2/`](../archive/sigma.mem_0.2/) — design chronicle, old API reference

## Docs
- Register a new document: `python3 ../tools/doc-register.py --id <ID> --type <type> --owner sigma.memory --path "sigma.memory/subdir/file.md" --title "..." --status open --tags "tag1,tag2"`
- `--type` values: `bug-report`, `feature-req`, `ticket`, `use-case`, `backlog`
- Or edit [q-or/registry.aml](../q-or/registry.aml) directly; full conventions: [q-or/DOC_STANDARDS.md](../q-or/DOC_STANDARDS.md)

## Ops
```
cbuild           # compile sources
cbuild lib       # compile + link shared lib
cbuild install   # install to /usr/local/packages/
cbuild clean     # remove build/ artifacts
./rtest          # run all tests
./rtest unit     # all tests in test/unit/
./rtest unit/<name>
```

### Dependencies
- Runtime: `sigma.core`, `sigma.collections`
- cbuild/rtest are at `/usr/local/scripts/`; `./rtest` is the local wrapper
- `sc_allocator_i` interface definition lives in `sigma.core` (not this library)
